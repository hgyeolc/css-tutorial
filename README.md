## 1. CSS ?

CSS(Cascading Stylesheet, 종속형 스타일시트)는 문서(document)의 스타일과 레이아웃을 정합니다.

- 제목 또는 컨텐츠의 글꼴, 색상, 크기 및 간격 변경
- 여러 개의 열로 분할
- 애니메이션이나 기타 장식 효과 추가

문서는 일반적으로 마크업 언어를 사용해 구성된 텍스트 파일입니다. HTML이 가장 일반적인 마크업 언어이지만, SVG 또는 XML과 같은 다른 마크업 언어를 사용할 수도 있습니다.

브라우저(Firefox, Chrome, Edge, ..)는 컴퓨터 화면, 프로젝터와 같은 곳에 문서를 시각적으로 표시합니다. 각 브라우저는 CSS를 사용하지 않아도 자체적으로 기본 스타일을 제공하며, 브라우저마다 조금씩 다릅니다.

브라우저는 사용자 에이전트(user agent)라고도 하며 컴퓨터 시스템에서 하나의 '개인'을 나타내는 프로그램을 의미합니다. CSS에서 사용자 에이전트는 주로 브라우저를 뜻하지만, 브라우저가 유일한 사용자 에이전트는 아닙니다.

### 1-1. CSS 구문

CSS는 웹 페이지의 특정 요소 또는 요소 그룹에 적용할 스타일 그룹을 지정합니다.

예를 들어 주요 제목을 큰 빨간색 텍스트로 지정하려면 다음과 같이 작성합니다.

```css
h1 {
  color: red;
  font-size: 5em;
}
```

- CSS 규칙은 위의 예시처럼 선택자(selector)로 시작됩니다. 선택자는 스타일을 지정할 HTML 요소를 선택합니다. 이 경우 `h1`을 스타일링합니다.
- 이후 중괄호 `{ }`가 열립니다.
- 중괄호 안에는 속성(property)과 값(value)이 쌍으로 이루어진 하나 이상의 선언(declaration)이 있습니다. 콜론 `:`을 기준으로 앞에는 속성(`color`), 뒤에는 값(`red`)을 명시합니다.

CSS 속성은 지정되는 속성에 따라 허용되는 값이 다릅니다. 위 예제의 `color`는 색상 값, `font-size`는 크기 단위를 값으로 사용할 수 있습니다.

## 2. CSS 작동 방식

### 2-1. 브라우저의 문서 처리 절차

브라우저가 문서를 표시할 때 아래와 같은 절차를 따릅니다. 다만 이해를 돕기 위해 매우 단순화시킨 것이며, 어떤 브라우저는 다른 방식으로 처리하기도 합니다. 하지만 대략적인 내용은 다음과 같습니다.

1. 브라우저가 네트워크에서 HTML 파일을 로드합니다.
2. HTML을 읽고 DOM(Document Object Model)으로 변환합니다. DOM은 컴퓨터 메모리에 적재된 문서를 나타냅니다.
3. 브라우저는 HTML 문서에 연결된 임베디드 이미지, 동영상, 링크된 CSS 등의 리소스들을 가져옵니다.
4. 브라우저가 CSS를 파싱하고 선택자 유형별로 분류합니다. 요소, 클래스, 아이디 등을 기반으로 DOM의 어느 노드에 어떤 규칙을 적용할지 결정합니다. 이 단계를 렌더 트리(render tree)라고 합니다.
5. 렌더 트리는 모든 CSS 규칙이 적용된 후에 표시되어야 하는 구조로 배치됩니다.
6. 페이지가 화면에 시각적으로 표시됩니다. 이 단계를 페인팅(painting)이라고 합니다.

### 2-2. DOM

DOM(Document Object Model)은 HTML 문서를 객체화시켜 나타낸 것을 말합니다. 트리 구조를 가지며 각 요소, 속성, 텍스트는 DOM의 노드(node)가 됩니다. 노드끼리는 부모, 자식, 형제의 관계가 형성됩니다.

### 2-3. 브라우저의 CSS 인식

모든 브라우저가 최신 CSS를 구현하는 것은 아닙니다. 또한 많은 사람들이 최신 버전의 브라우저를 사용하지 않습니다. CSS는 항상 새로운 기술이 개발되고 있으므로 브라우저가 인식하지 못하는 선택자 또는 선언을 만날 수도 있습니다.

위처럼 브라우저가 이해하지 못하는 규칙을 만나면, 브라우저는 이를 무시하고 다음 선언으로 넘어갑니다. 오류 발생, 철자 오타, 속성 또는 값이 상대적으로 최신이여서 아직 이를 지원하지 않는 경우 등이 해당됩니다.

## 3. 선택자 종류

### 3-1. 기본 선택자

- `*`: 모든 요소를 선택합니다.
- `A`: 태그 이름이 A인 요소를 선택합니다.
- `.A`: 클래스 속성이 A인 요소를 선택합니다.
- `#A`: 아이디 속성이 A인 요소를 선택합니다.

### 3-2. 복합 선택자

- `AB`: A, B를 동시에 만족하는 요소를 선택합니다.
- `A > B`: A의 직계 자식인 B 요소를 선택합니다.
- `A B`: A의 하위 요소 중 B 요소를 선택합니다.
- `A + B`: A의 다음 형제 요소 중 B 한 개를 선택합니다.
- `A ~ B`: A의 다음 형제 요소 중 B 모두를 선택합니다.

### 3-3. 가상(Pseudo) 클래스 선택자

- `A:hover`: A에 커서가 올려져 있는 동안 선택합니다.
- `A:active`: A를 클릭하고 있는 동안 선택합니다.
- `A:focus`: A가 포커스(focus, 활성화) 되면 선택합니다. 포커싱이 가능한 요소는 HTML 대화형 컨텐츠인 `<input>`, `<a>`, `<button>` 등이 있습니다. 본래 포커싱이 가능한 요소가 아니지만 `tabindex` 속성으로도 포커싱이 가능하도록 만들 수 있습니다. `tabindex`의 값은 `-1`을 권장합니다.
- `A:first-child`: A임과 동시에 형제 요소 중 첫째라면 선택합니다.
- `A:last-child`: A임과 동시에 형제 요소 중 막내라면 선택합니다.
- `A:nth-child(n)`: A임과 동시에 형제 요소 중 `n`째라면 선택합니다.
- `A:not(B)`: A이면서 B는 아닌 요소를 선택합니다.

### 3-4. 가상(Pseudo) 요소 선택자

- `A::before`: A의 첫째 자식으로 텍스트 문자열을 삽입합니다. `content` 속성은 필수로 명시해야하고 속성의 값(삽입 내용)이 없다면 공백으로 작성합니다. CSS에서 텍스트 문자열을 삽입하는 것은 자주 있는 일은 아닙니다. 해당 텍스트는 일부 스크린 리더에서는 접근할 수 없고, 코드를 보는 다른 개발자에게도 혼란을 주기 때문입니다. 이 선택자는 요소 사이에 수직 구분선을 나타낼 때 주로 사용합니다. 빈 문자열을 추가하고 `display: block`으로 설정하여 스타일을 지정할 수 있도록 하는 방식입니다.
- `A::after`: A의 막내 자식으로 내용을 삽입합니다. 이후 설명은 위와 동일합니다.
- `A::first-line`: A의 첫 번째 행만 선택합니다.

### 3-5. 속성(Attribute) 선택자

- `[A]`: 속성 A를 가진 요소를 선택합니다. 속성의 값을 특정하고 싶다면 `[A="B"]`로 작성합니다.

## 4. 스타일 상속(Inheritance)과 선택자 우선순위(Specificity)

### 4-1. 스타일 상속

부모 요소가 가진 속성을 하위 요소가 상속 받습니다. 모든 요소가 상속되지는 않으며 주로 글자와 관련된 속성들이 상속됩니다.

- `font-style`
- `font-weight`
- `font-size`
- `font-family`
- `color`
- `line-height`
- `text-align`

속성 값이 자주 변경될 경우 `inherit` 값을 사용하면 부모 요소의 속성을 자동으로 물려받도록 할 수 있습니다. 이를 강제상속이라고 합니다.

### 4-2. 선택자 우선순위

- 1점: 요소(태그), 가상 요소 선택자
- 10점: 클래스, 가상 클래스, 속성 선택자
- 100점: 아이디 선택자
- 1000점: 인라인 선언
- !important : 모든 계산을 무효화한 최우선순위. 반드시 필요한 경우가 아니라면 사용하지 않는 것이 좋습니다.

전체 선택자(`*`), 결합자(`+`, `>`, `~`, 공백), 그리고 가상 클래스 선택자 중 `:not`은 우선순위에 영향을 주지 않습니다(0점).

우선순위를 나타낸 점수는 이해를 돕기 위한 예시일 뿐입니다. 실제로 각 선택자 유형에는 고유한 순위가 있으며, 낮은 순위의 선택자로 덮어쓸 수 없습니다. 예를 들어 백만개의 클래서 선택자가 결합되어도 하나의 아이디 선택자 규칙보다 우선시 되지 않습니다.

## 5. 박스(Box) 모델

### 5-1. 블록(Block) 및 인라인(Inline) 박스

CSS에는 크게 블록과 인라인 박스 유형이 있습니다. 블록 박스는 다음 방식으로 동작합니다.

- 박스의 너비가 상위 컨테이너에서 사용 가능한 공간을 모두 채웁니다.
- 줄바꿈이 일어납니다.
- `width`와 `height` 속성의 값이 적용됩니다.
- 내부 여백(`padding`), 외부 여백(`margin`), 테두리(`border`)를 지정하면 그로 인해 다른 요소들이 박스로부터 밀려납니다.

인라인 박스는 다음 방식으로 동작합니다.

- 줄바꿈이 일어나지 않습니다.
- `width`와 `height` 속성에 값을 넣어도 적용되지 않습니다.
- 좌우의 내/외부 여백(상하 여백 제외), 테두리를 지정하면 그로 인해 다른 인라인 요소들이 박스로부터 밀려납니다.

### 5-2. 박스 모델

페이지에서 나타날 박스의 여백, 테두리, 컨텐츠 등이 어떻게 함께 작동할 것인지를 정의합니다. 박스 구성은 다음과 같습니다.

- Content Box: 컨텐츠가 표시되는 영역입니다. `width`, `height`와 같은 속성으로 지정합니다.
- Padding Box: 컨텐츠 주변을 공백과 같이 차지합니다. `padding` 속성으로 제어합니다.
- Border Box: 컨텐츠와 내부 여백을 포함합니다. `border` 속성으로 제어합니다.
- Margin Box: 가장 바깥 레이어(layer)로 컨텐츠, 내부 여백, 테두리를 감싸며 다른 요소 사이의 공백 역할을 합니다. `margin` 속성으로 제어합니다.

외부 여백(margin)은 박스의 실제 크기에 포함되지 않습니다. 즉, 박스의 영역은 테두리까지입니다. 외부 여백은 말 그대로 박스의 외부 공간에만 영향을 미칩니다.

## 6. CSS 속성(Property)

다음은 주로 사용되는 속성들입니다.

### 6-1. 너비

- `width`, `height`
  - 요소의 가로(width), 세로(height) 너비를 지정합니다.
  - `auto`: (기본값) 브라우저가 너비를 자동으로 계산합니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.
- `max-width`, `max-height`
  - 요소의 최대 가로, 세로 너비를 지정합니다. 요소는 작성된 것보다 큰 너비를 가질 수 없습니다.
  - `none`: (기본값) 최대 너비를 제한하지 않습니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.
- `min-width`, `min-height`
  - 요소의 최소 가로, 세로 너비를 지정합니다. 요소는 작성된 것보다 작은 너비를 가질 수 없습니다.
  - `0`: (기본값) 최소 너비를 제한하지 않습니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.

### 6-2. 외부 여백(Margin)

- `margin`
  - 요소의 외부 여백을 지정합니다.
  - 단축 속성입니다(top, right, bottom, left).
  - 음수를 사용할 수 있습니다.
  - `0`: (기본값) 외부 여백을 지정하지 않습니다.
  - `auto`: 브라우저가 여백을 계산합니다. 가로(세로) 너비가 있는 요소의 가운데 정렬에 활용할 수 있습니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.

### 6-3. 내부 여백(Padding)

- `padding`
  - 요소의 내부 여백을 지정합니다.
  - 단축 속성입니다.
  - 지정하면 요소의 크기가 커지는 특징이 있습니다.
  - `0`: (기본값) 내부 여백을 지정하지 않습니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.
  - `%`: 부모 요소의 가로 너비에 대한 비율로 지정합니다.

### 6-4. 테두리(Border)

- `border`
  - 요소의 테두리 선을 지정합니다.
  - 단축 속성입니다(width, style, color / top, right, bottom, left).
  - 지정하면 요소의 크기가 커지는 특징이 있습니다.
  - 일반적으로 `border: 두께 종류 색상` 순으로 작성합니다.
- `border-width`
  - 테두리 선 두께를 지정합니다.
  - 단축 속성입니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.
- `border-style`
  - 테두리 선 종류를 지정합니다.
  - 단축 속성입니다.
  - `none`: (기본값) 선을 지정하지 않습니다.
  - `solid`: 실선을 지정합니다.
  - `dashed`: 파선을 지정합니다.
- `border-color`
  - 테두리 선 색상을 지정합니다.
  - 단축 속성입니다.
  - `black`: (기본값) 검정색으로 지정합니다.
  - 색상: 다양한 색상 표현을 이용해 지정합니다.
  - `transparent`: 투명하게 만듭니다.
- `border-radius`
  - 요소의 모서리를 둥글게 깎습니다.
  - 지정한 길이만큼의 반지름을 갖는 원을 모서리에 그린 후 원 바깥 부분을 깎아냅니다.
  - 각 모서리를 따로 지정할 수 있습니다. 왼쪽 위부터 시계 방향 순서입니다.
  - `0`: (기본값) 모서리를 깎지 않습니다.
  - `px`, `em`, `vw` 등: 적절한 단위를 사용해 작성합니다.

### 6-5. 박스 크기 계산(Box Sizing)

- `box-sizing`
  - 요소의 크기 계산 기준을 지정합니다.
  - `content-box`: (기본값) 내용(content)으로 크기를 계산합니다.
  - `border-box`: 내용 + 내부 여백 + 테두리로 크기를 계산합니다.

### 6-6. 넘침 제어(Overflow)

- `overflow`
  - 요소의 크기 이상으로 내용이 넘쳤을 때, 보여짐을 제어합니다.
  - 단축 속성입니다(x, y).
  - `visible`: (기본값) 넘친 내용을 그대로 표시합니다.
  - `hidden`: 넘친 내용을 잘라냅니다.
  - `auto`: 내용이 넘쳤을 경우에만 잘라낸 후 스크롤바를 생성합니다.

### 6-7. 출력 특성(Display)

- `display`
  - 요소의 화면 표시(보여짐) 특성을 지정합니다.
  - `inline`: (기본값) 인라인 요소로 표시합니다.
  - `block`: (기본값) 블록 요소로 표시합니다.
  - `inline-block`: (기본값) 인라인-블록 요소로 표시합니다. 기본적으로 인라인 요소의 특성을 가지지만 너비와 여백을 지정할 수 있습니다.
  - `flex`: 규칙이 선언된 요소는 플렉스 컨테이너(flex container)가 되며 자식 요소들을 1차원 레이아웃으로 표시합니다.
  - `inline-flex`: 플렉스 컨테이너를 인라인 요소로 취급하며 정의합니다.
  - `grid`: 2차원 레이아웃으로 표시합니다.
  - `none`: 화면에서 보이지 않게 만듭니다.

### 6-8. 투명도(Opacity)

- `opacity`
  - 요소의 투명도를 지정합니다.
  - `1`: (기본값) 불투명하게 표시합니다.
  - `0 ~ 1`: 0부터 1사이의 소수점 숫자로 작성합니다. 0은 투명, 1은 불투명함을 의미합니다.

### 6-9. 글꼴

- `font-style`
  - 글자의 기울기를 지정합니다.
  - `normal`: (기본값) 기울기 없이 표시합니다.
  - `italic`: 이탤릭체로 표시합니다.
- `font-weight`
  - 글자의 두께(가중치)를 지정합니다.
  - `normal, 400`: (기본값) 기본 두께로 표시합니다.
  - `bold, 700`: 두껍게 표시합니다.
  - `100 ~ 900`: 100에서 900 사이의 백단위 숫자 9개 중 하나를 지정합니다. 900에 가까울수록 두껍게 표시됩니다.
- `font-size`
  - 글자의 크기를 지정합니다.
  - `16px`: (기본값) 기본 크기입니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다.
- `line-height`
  - 한 줄의 높이를 지정합니다.
  - `normal`: (기본값) 브라우저가 정의한 기본 높이를 사용합니다.
  - 숫자: 요소 글꼴 크기의 배수로 지정합니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다.
- `font-family`
  - 요소의 글꼴을 지정합니다.
  - 글꼴 후보를 `,`로 구분해 나열합니다. 글꼴 이름에 공백이나 특수문자가 들어갈 경우 `""`로 묶어 작성합니다.
  - (필수값) 모든 글꼴 후보가 사용 불가능할 경우를 대비해 마지막에 글꼴계열을 명시합니다. 해당 계열 중 하나로 적용됩니다.
  - `글꼴1, "글꼴2", ... 글꼴 계열`과 같이 작성합니다. 대표적인 글꼴 계열로는 `sans-serif`(고딕체), `serif`(바탕체)가 있습니다.

### 6-10. 문자

- `color`
  - 글자의 색상을 지정합니다.
  - `rgb(0, 0, 0)`: (기본값) 검정색으로 표시합니다.
  - 색상: 기타 지정 가능한 색상을 작성합니다.
- `text-align`
  - 글자의 정렬 방식을 지정합니다.
  - `left`: (기본값) 왼쪽으로 정렬합니다.
  - `right`: 오른쪽으로 정렬합니다.
  - `center`: 가운데로 정렬합니다.
- `text-decoration`
  - 글자의 장식을 지정합니다.
  - `none`: (기본값) 장식 없이 표시합니다.
  - `underline`: 밑줄을 표시합니다.
  - `line-through`: 중앙선(취소선)을 표시합니다.
- `text-indent`
  - 첫 줄의 들여쓰기를 지정합니다.
  - 내어쓰기(outdent)는 음수를 지정해 사용할 수 있습니다.
  - `0`: (기본값) 들여쓰기 없이 표시합니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다.

### 6-11. 배경

- `background-color`
  - 요소의 배경 색상을 지정합니다.
  - `transparent`: (기본값) 투명한 배경을 적용합니다.
  - 색상: 지정 가능한 색상을 작성합니다.
- `background-image`
  - 요소의 배경 이미지를 삽입합니다.
  - `none`: (기본값) 배경 이미지를 삽입하지 않습니다.
  - `url("경로")`: 이미지 경로를 작성합니다.
- `background-repeat`
  - 요소의 배경 이미지 반복을 지정합니다.
  - `repeat`: (기본값) 이미지를 수직, 수평으로 반복합니다.
  - `repeat-x`: 이미지를 수평 반복합니다.
  - `repeat-y`: 이미지를 수직 반복합니다.
  - `no-repeat`: 이미지를 반복하지 않습니다.
- `background-position`
  - 요소의 배경 이미지 위치를 지정합니다.
  - `top`, `bottom`, `left`, `right`, `center`: 방향을 지정합니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다. x, y축에 해당하는 두 위치를 입력합니다.
- `background-size`
  - 요소의 배경 이미지 크기를 지정합니다.
  - `auto`: (기본값) 이미지의 실제 크기로 지정합니다.
  - `cover`: 비율을 유지하며 요소의 더 긴 너비에 맞춥니다.
  - `contain`: 비율을 유지하며 요소의 더 짧은 너비에 맞춥니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다.
- `background-attachment`
  - 요소의 배경 이미지 스크롤 특성을 지정합니다.
  - `scroll`: (기본값) 이미지가 요소를 따라 같이 스크롤됩니다.
  - `fixed`: 이미지가 뷰포트에 고정 되어 스크롤 되지 않습니다.

### 6-12. 배치

- `position`
  - 요소의 위치 기준을 지정합니다.
  - `top`, `bottom`, `left`, `right`, `z-index` 등의 속성과 함께 사용합니다.
  - 음수를 사용할 수 있습니다.
  - `static`: (기본값) 기준 없이 배치합니다.
  - `relative`: 요소 자신을 기준으로 배치합니다. 이때 요소는 실제로 이동한 것이 아닌 시각적으로만 이동됩니다. 요소 자신을 기준으로 정하는 경우보다는 `absolute`의 위치상 부모 요소를 지정할 때 주로 사용합니다.
  - `absolute`: 위치상 부모 요소(`relative`)를 기준으로 배치합니다. 더이상 주변 요소와 상호작용하지 않습니다. 부모 요소 중 `relative` 요소를 찾지 못할 경우 최종적으로 뷰포트를 기준으로 배치합니다. 요소의 `display` 속성이 `block`으로 변경됩니다.
  - `fixed`: 뷰포트를 기준으로 배치합니다. 더이상 주변 요소와 상호작용하지 않습니다. 요소의 `display` 속성이 `block`으로 변경됩니다.
  - `sticky`: 지정한 영역에 도달하기 전까지는 스크롤 되다가 도달 후 `fixed`처럼 동작합니다.

### 6-13. 요소 쌓임 순서(Stack Order)

요소의 쌓임은 다음과 같은 우선순위를 가집니다.

1. 요소에 `position` 속성의 값이 있는 경우 위에 쌓입니다(기본값 `static` 제외).
2. 1번 조건이 동일할 때, `z-index` 속성의 값이 높을수록 위에 쌓입니다.
3. 1,2번 조건이 동일할 때, HTML에서 더 나중에 생성된 요소일수록 위에 쌓입니다.

- `z-index`
  - 요소의 쌓임 정도를 지정합니다.
  - 음수를 사용할 수 있지만 `-1` 정도를 제외하고는 거의 사용되지 않습니다.
  - `auto`: (기본값) 부모 요소와 동일한 쌓임 정도를 갖습니다.
  - 숫자: 숫자가 높을수록 위에 쌓입니다.

### 6-14. 1차원 레이아웃(Flex)

플렉스 컨테이너에 부여하는 속성들은 다음과 같습니다.

- `flex-direction`
  - 주 축(main axis)을 지정합니다.
  - `row`: (기본값) 가로축을 주 축으로 지정합니다(좌 -> 우).
  - `row-reverse`: 가로축을 주 축으로 지정합니다(우 -> 좌).
  - `column`: 세로축을 주 축으로 지정합니다(상 -> 하).
  - `column-reverse`: 세로축을 주 축으로 지정합니다(하 -> 상).
- `flex-wrap`
  - 플렉스 아이템(flex-item)이 컨테이너 밖으로 넘칠 때 묶음(줄바꿈) 여부를 지정합니다.
  - `nowrap`: (기본값) 줄바꿈 없이 배치합니다. 아이템이 찌그러집니다.
  - `wrap`: 공간이 부족할 경우 줄바꿈하여 배치합니다. 아이템이 찌그러지지 않습니다.
- `justify-content`
  - 주 축의 정렬 방법을 지정합니다.
  - `flex-start`: (기본값) 아이템을 시작점으로 정렬합니다.
  - `flex-end`: 아이템을 끝점으로 정렬합니다.
  - `center`: 아이템을 가운데 정렬합니다.
  - `space-between`: 아이템 사이를 균등하게 정렬합니다.
  - `space-around`: 아이템의 외부 여백을 균등하게 정렬합니다.
- `align-content`
  - 교차 축(cross axis)에서의 여러 줄 정렬 방법을 지정합니다.
  - `stretch`: (기본값) 아이템을 각 줄의 시작점에 정렬합니다.
  - `flex-start`: 아이템을 첫 줄의 시작점에 정렬합니다.
  - `flex-end`: 아이템을 첫 줄의 끝점에 정렬합니다.
  - `center`: 아이템을 가운데 정렬합니다.
  - `space-between`: 아이템 사이를 균등하게 정렬합니다.
  - `space-around`: 아이템의 외부 여백을 균등하게 정렬합니다.
- `align-items`
  - 교차 축의 한 줄 정렬 방법을 지정합니다.
  - `stretch`: (기본값) 아이템을 교차 축으로 늘립니다.
  - `flex-start`: 아이템을 각 줄의 시작점에 정렬합니다.
  - `flex-end`: 아이템을 각 줄의 끝점에 정렬합니다.
  - `center`: 아이템을 각 줄의 가운데에 정렬합니다.

플렉스 아이템에 부여하는 속성들은 다음과 같습니다.

- `order`
  - 아이템의 순서를 지정합니다.
  - `0`: (기본값) 순서를 정하지 않습니다.
  - 숫자: 숫자가 작을수록 먼저 배치됩니다.
- `flex-grow`
  - 아이템의 너비 증가 비율을 지정합니다.
  - `0`: (기본값) 증가 비율이 없음을 뜻합니다.
  - 숫자: 숫자만큼 증가 비율을 적용합니다.
- `flex-shrink`
  - 아이템의 너비 감소 비율을 지정합니다.
  - `1`: (기본값) 컨테이너 너비에 따라 감소 비율을 적용합니다.
  - 숫자: 숫자만큼 감소 비율을 적용합니다.
- `flex-basis`
  - 아이템의 공간 배분 전 기본 너비를 지정합니다.
  - `auto`: (기본값) 요소의 컨텐츠 너비만큼 적용합니다.
  - `px`, `em`, `rem` 등: 적절한 단위를 사용해 작성합니다.

### 6-15. 전환(Transition)

- `transition`
  - 요소의 전환(시작과 끝) 효과를 지정합니다.
  - 단축 속성입니다(속성명, 지속시간, 타이밍 함수, 대기시간).
  - 단축형으로 작성할 때 지속시간은 필수로 작성해야 합니다.
